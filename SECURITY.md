# Security policy

CyberWiki treats security as a core requirement for any deployment that manages documentation about cybernetics, BCI, EEG, and nanoswarm systems. The Rust workspace is developed using current best practices for dependency management, input validation, and least-privilege design.[web:111][web:114]

Security-sensitive changes should be reviewed carefully, avoiding unnecessary use of unsafe code and ensuring that external dependencies remain up to date. Vulnerabilities or suspected issues can be reported through the repository’s issue tracker or designated contact channels, following any additional guidance provided on the project homepage.[web:111][web:114]
