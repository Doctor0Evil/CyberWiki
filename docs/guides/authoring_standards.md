# Authoring standards

All CyberWiki content must be written in precise, non-fictional prose that describes real systems, protocols, datasets, and governance frameworks. Descriptions should be reproducible and tied to verifiable sources, avoiding speculative predictions, metaphors, or promotional language.[web:20][web:56][web:57]

Authors should normalize terminology to recognized standards in mathematics, geography, geology, and biophysics when these disciplines intersect with cybernetic and brain–computer systems. Example descriptions and function summaries should emphasize operational behavior, measurable parameters, and integration boundaries instead of high-level marketing language or anecdotal narratives.
